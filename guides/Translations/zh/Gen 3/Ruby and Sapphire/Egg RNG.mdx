## 所需工具

- [带有 Lua 脚本的 mGBA 模拟器](/mgba-setup)
- [PokeFinder](/pokefinder)
- 父母个体值与相性（向培育屋老爷爷询问）
- 表 ID 与里 ID（如需孵化闪光个体）

## 简介

在红宝石/蓝宝石中，蛋的PID是分为两部分生成的。蛋生成时会决定 PID 的前半部分，领取蛋时决定后半部分。

红宝石/蓝宝石的蛋乱数在实电池与干电池环境下都可以进行。

```
注意：在红宝石与蓝宝石中，持有不变之石并不能遗传性格。
```

## 准备阶段

1. 将宝可梦交给培育屋。请确保父母都未携带道具，放入顺序没有影响。
2. 待在培育屋内行走，直到 Lua 脚本显示步数计数为 FE。必须在培育屋内部完成这一步。
3. 达成条件后，在此处建立即时存档点。

![Setup](/images/Ruby-Sapphire/Egg/Setup.png)

## PokeFinder 设置

1. 打开 PokeFinder，选择第 3 世代孵化乱数，并确保切换到红宝石/蓝宝石的标签页。
2. 输入双亲的所有信息，包括他们的相性。
3. 帧数范围可以自由设定，但“生成帧”（Frame Held）上限必须低于“领取帧”（Pickup Frame）下限。
4. 输入 Lua 脚本中显示的初始种子，开始查找个体组合。如果是干电池，红宝石/蓝宝石的初始种子固定为 5A0。

![Initial Seed](/images/Ruby-Sapphire/Egg/Initial-Seed.png)

5. 找到目标组合后，请注意需要命中两个不同的帧数。

## 生成帧乱数

1. 在正确的帧数上向前走一步以生成蛋。在模拟器暂停时可使用 CTRL + N 一帧一帧推进。当推进到目标帧数时，按住方向键并取消暂停。对于 PokeFinder，需要从目标帧数中减去 18，在这个帧数上行走一步。如果未生成蛋，请确认输入信息并尝试使用延迟 17 或 19。
2. 行走后，脚本会显示 PID 的后半部分。如果匹配目标，就可以继续；若不匹配，请重启模拟器或尝试下一个生成帧。

![Held Frame](/images/Ruby-Sapphire/Egg/Held.png)

3. 至此，生成帧的乱数已完成。接下来是领取帧的乱数。

## 领取帧乱数

1. 为防止错过按键时机，建议先建立一个即时存档点。
2. 离开培育屋，与老爷爷对话并接受蛋。在出现最后一句对话（“请好好照顾它。”）时暂停模拟器。
3. 推进到目标帧数，暂停，按住 A 键并取消暂停。领取帧的标准延迟为 3。如果蛋的 PID 不正确，可尝试延迟 2 或 4。

这样你就会获得目标 PID 的第二部分，从而完成整个蛋乱数。

![Success](/images/Ruby-Sapphire/Egg/Success.png)

若结果不符，请重新检查整个流程，并善用即时存档点进行尝试。
